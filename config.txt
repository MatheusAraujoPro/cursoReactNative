CRIANDO O PROJETO
    |- Dentro da pasta do projeto criei uma nova pasta chamda backend
        |-- backend: iniciei o projeto através do comando yarn init -yarn

        yarn init -yarn 
            |- Cria o projeto através do package.json
                |-package.json
                    |-Armazena algumas informações cruciais sobre o app e as dependências

        Insalando dependências
        |-> yarn add nomeDaDependecia


        Midleware
        |-> função que recebe dois parâmetros(req, res) e devolve um valor
        |-> DESENHO: (req, res)=>{}
        |-> Caso eu queria usar variáveis dentro das strings, substituo as aspas simples 
            pelos apostrofos
        
        Nodemom
        |-> Dependência de desenvolvimento
        |-> Crio um "Novo perfil"de desenvolvimento
        |-> Restarta o servidor sempre que há uma alteração

        Multer
        |->É uma biblioteca que lida com o corpo de requisições mult part form data
        |->Automatiza o processo de upload dentro de arquivos dentro backend

        Sharp
        |-> Permite manipulação de imagens
        |-> É necessário redimencionar a imagem devido ao tamamho padrão usado no mobile

        Cors
        |-> Dependência que permite que o backend possa ser acessível pelo front mesmo ele 
            estando em domínios diferentes

        Obs: O Protoclo de comunicação real time não é suportado pelo Nodemom
        no caso de eu qeurer usar terei que baixar mais uma dependência o socket.informações
